## Privacy and responsibility

Self-hosting changes who is responsible for your data. Renalo does not send your budgeting records to a Renalo-operated cloud, but your deployment operator can access the database and application logs. Protect the host, use HTTPS, keep the JWT secret private, restrict database access, and maintain tested backups.

Passkeys, remembered sessions, activation links, and sign-in links all depend on the security of your deployment. Treat `RENALO_PUBLIC_URL` as stable configuration: changing the public origin can make registered passkeys unusable.
