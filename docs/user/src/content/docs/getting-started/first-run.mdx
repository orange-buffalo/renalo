## Capture the generated administrator

When the database contains no administrator, Renalo creates one and writes a single boxed log entry:

```text
============================================================
Renalo default admin user created
------------------------------------------------------------
Username: admin
Password: <generated-password>
------------------------------------------------------------
Store these credentials now. The generated password is only
logged at creation time.
============================================================
```

With Compose, find it using:

```bash
docker compose logs renalo
```

The username can be changed before first start with `RENALO_DEFAULT_ADMIN_USERNAME`. This variable does not rename or reset an existing administrator.

## Create a regular user

1. Sign in as the generated administrator.
2. Open **User management**.
3. Select **Create user**.
4. Choose the regular user type and enter a username.
5. Open the newly created user and copy the activation link.
6. Send that link through a trusted channel.

New users are inactive until they set a password through the activation page. Activation links expire after 24 hours and can be regenerated by an administrator.

## Prepare the budget workspace

After the regular user signs in:

1. Open **Budget settings** and review the default `Main` account.
2. Set the correct currency and opening balance before recording activity.
3. Add any additional bank, cash, savings, or foreign-currency accounts.
4. Create useful expense and income categories.
5. Record transactions manually or use the [Toshl importer](/features/toshl-import/).

The opening balance is the balance immediately before the activity you intend to record. Do not include the same historical activity both in the opening balance and as imported transactions.

## Secure each account

Users can register passkeys from **My profile**, generate a five-minute sign-in link for another device, and optionally disable password sign-in once at least one passkey exists. Keep more than one usable sign-in method or involve an administrator before removing the final passkey.
