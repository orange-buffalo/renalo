Administrators manage local Renalo users. Renalo has no external identity-provider requirement and no public self-registration flow.

## Create a user

Choose a username and either regular-user or administrator type. New users are inactive and receive no preset password.

<ScreenshotPair desktop={createDesktop} mobile={createMobile} alt="Create Renalo user form" />

Open the inactive user and copy the activation link. It expires after 24 hours; regenerate it when necessary.

<ScreenshotPair desktop={inactiveDesktop} mobile={inactiveMobile} alt="Inactive user activation controls" />

## Operational guidance

- Send activation links through a trusted channel.
- Deactivate or remove access promptly when a user leaves.
- User deletion is destructive and can cascade through user-owned records.
- Do not delete an administrator to recover access; follow the [administrator recovery procedure](/getting-started/admin-recovery/).
- Authentication changes may not immediately invalidate already-issued 30-minute JWTs. Revoke remembered sessions and rotate the JWT secret during a security incident.
